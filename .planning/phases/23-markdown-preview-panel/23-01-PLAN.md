  <action>
Create `src/components/results/MarkdownPreviewPanel.vue` as a full-screen modal overlay that:

1. **Props**: `open: boolean` (controls visibility)
2. **Emits**: `close` (when user clicks close button or backdrop)
3. **Script setup** (Vue 3 Composition API with TypeScript):
   - Import `ref, watch` from `vue`
   - Import `useI18n` from `vue-i18n`
   - Import `generateMarkdownReport` from `@/composables/useMarkdownExport`
   - Declare module-level cached variables for lazy-loaded modules (NOT top-level imports):
     ```typescript
     let cachedMarked: typeof import('marked') | null = null
     let cachedPurify: typeof import('dompurify') | null = null
     ```
   - Reactive state: `htmlContent = ref('')`, `loading = ref(false)`
   - `loadPreview()` async function:
     - Sets `loading.value = true`
     - Calls `generateMarkdownReport()` to get the Markdown string
     - Lazy-loads marked via `cachedMarked = cachedMarked ?? await import('marked')`
     - Lazy-loads dompurify via `cachedPurify = cachedPurify ?? await import('dompurify')`
     - Parses: `const rawHtml = await cachedMarked.marked.parse(md)`
     - Sanitizes: `htmlContent.value = cachedPurify.default.sanitize(rawHtml)`
     - Sets `loading.value = false`
     - Wraps the above in try/finally to ensure loading is reset on error
   - `watch(() => props.open, (isOpen) => { if (isOpen) loadPreview() })`

4. **Template**:
   - Outer `<div v-if="open">` with classes: `fixed inset-0 z-50 flex items-start justify-center bg-black/50 print:hidden`
   - Click on backdrop (the outer div) emits `close`
   - Inner content panel `<div>` with `@click.stop` to prevent backdrop click propagation, classes: `relative mt-8 mb-8 mx-4 max-w-4xl w-full max-h-[calc(100vh-4rem)] overflow-y-auto rounded-lg bg-white dark:bg-gray-900 shadow-xl`
   - Sticky header bar inside the panel: `<div class="sticky top-0 z-10 flex items-center justify-between px-6 py-3 bg-white dark:bg-gray-900 border-b border-gray-200 dark:border-gray-700">`
     - Title: `<h2 class="text-lg font-semibold text-gray-900 dark:text-gray-100">{{ t('results.toolbar.preview') }}</h2>`
     - Close button: `<button @click="emit('close')" class="px-3 py-1.5 text-sm font-medium rounded border border-gray-300 dark:border-gray-600 bg-white dark:bg-gray-800 text-gray-700 dark:text-gray-300 hover:bg-gray-50 dark:hover:bg-gray-700 transition-colors">{{ t('results.toolbar.previewClose') }}</button>`
   - Body: `<div class="px-6 py-4">`
     - Loading state: `<p v-if="loading" class="text-gray-500 dark:text-gray-400">...</p>`
     - Content: `<div v-else class="prose dark:prose-invert max-w-none" v-html="htmlContent" />`
   - The `prose dark:prose-invert` classes from @tailwindcss/typography handle all Markdown element styling (headings, tables, code blocks, lists)

5. **CRITICAL SECURITY**: The `v-html` binding MUST only receive output from `DOMPurify.sanitize()`. Never pass raw `marked.parse()` output directly to `v-html` (PITFALL-9).

6. **CRITICAL BUNDLE**: marked and dompurify MUST be loaded via dynamic `import()` inside `loadPreview()`, NOT via top-level static imports. Use the module caching pattern (cachedMarked/cachedPurify) so subsequent opens don't re-import (mirrors PPTX-15 pattern).
