## Spacing Scale

Standard 8-point scale. Pre-populated from existing codebase pattern (`space-y-4`, `gap-4`, `p-4` dominates all existing sections).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`px-2 py-1`) |
| sm | 8px | Button group gaps (`gap-2`), compact row spacing |
| md | 16px | Default element spacing (`gap-4`, `space-y-4`) |
| lg | 24px | Section padding top (border separator `pt-2` = 8px within sections) |
| xl | 32px | Layout column gap (split-screen) |
| 2xl | 48px | — (not used in input/result cards) |
| 3xl | 64px | — (page-level, not used in card scope) |

**Exceptions:**
- Profile dropdown and storage nodes slider: contained in `grid grid-cols-1 sm:grid-cols-2 gap-4` (16px), same as FTT/RAID grid in StorageConfigForm.
- Network speed button group: `flex gap-2 flex-wrap` (8px), same as existing type button groups.
- VsanMaxClusterCard data grid: `grid grid-cols-2 gap-x-4 gap-y-1` (16px / 4px), same as HostCountCard details grid.

Source: codebase scan of existing components.

## Typography

Pre-populated entirely from codebase scan. All existing components use the same 4-role scale.
**Weight rule: exactly 2 weights — `font-semibold` (600) for all emphasis roles, `font-normal` (400) for all default roles.**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Section heading | 16px (`text-base`) | 600 (`font-semibold`) | 1.5 | `<h2>` in form sections and result cards |
| Card sub-heading | 14px (`text-sm`) | 600 (`font-semibold`) | 1.5 | Column headers in data grids (`text-sm font-semibold text-gray-500`) |
| Body / label | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | Form labels, button group labels, informational notes |
| Data / mono | 12px (`text-xs`) | 400 (`font-normal`) + 600 (`font-semibold`) for values | 1.5 | Data grid cells (`font-mono text-right`), capacity values |

**New in Phase 5 — VsanMaxClusterCard large display number:**
- Storage node count: 48px (`text-5xl`), weight 600 (`font-semibold`), line-height none (`leading-none`) — mirrors HostCountCard's `recommendedHostCount` display. The 48px size provides sufficient visual distinction without needing `font-bold`.

Source: codebase scan — HostCountCard.vue, StretchNetworkChecklist.vue, StorageConfigForm.vue.

## Color

Pre-populated from codebase scan. Project uses Tailwind CSS v4 with dark mode (`dark:` variants throughout).

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `white` / `gray-50` | `gray-900` | Page background, form section backgrounds |
| Secondary (30%) | `white` / `gray-50` | `gray-800` | Card backgrounds (`bg-white dark:bg-gray-800`) |
| Border | `gray-200` | `gray-700` | Card borders (`border-gray-200 dark:border-gray-700`) |
| Accent (10%) | `blue-600` | `blue-400` | Active state on button groups, focus rings, mono data values |
| Success | `green-700` | `green-400` | Usable capacity value in storage summary, sufficient host count |
| Error | `red-600` | `red-300` | WarningBanner severity=error, below-minimum node count indicator |
| Warning | `yellow-800` | `yellow-300` | WarningBanner severity=warning (dedup network speed warning) |
| Info | `blue-700` | `blue-400` | Info note boxes (`bg-blue-50 dark:bg-blue-900/30 border-blue-200`) |
| Muted text | `gray-500` | `gray-400` | Card sub-headings, data labels |

**Accent reserved for:**
- Active/selected state of all button group buttons (storage type, network speed)
- Focus ring on all interactive inputs (`focus:ring-blue-500`)
- Mono data values in data grids (`text-blue-700 dark:text-blue-400`)
- Slider track accent (`accent-blue-600`)

**Destructive:** Not applicable. Phase 5 has no destructive actions. Validation errors use `red-600 / red-300` via the existing WarningBanner component.

Source: codebase scan — all existing components.

## Interaction Contracts

### Storage type button group (StorageConfigForm)

| State | Visual |
|-------|--------|
| Active (vSAN Max selected) | `bg-blue-600 text-white border-blue-600` |
| Inactive | `bg-white dark:bg-gray-700 text-gray-700 border-gray-300 hover:border-blue-400` |
| Section reveal | vSAN Max profile dropdown + storage nodes slider appear instantly via `v-if` (no animation — matches existing pattern) |
| Section hide | When a different type is selected, vSAN Max section unmounts from DOM |

### Profile dropdown (StorageConfigForm)

| State | Visual |
|-------|--------|
| Default value | `med` (MED — 100 TB/node) |
| Focus | `focus:ring-2 focus:ring-blue-500` |
| No disabled options | All 5 profiles always available |

### Storage nodes slider (NumberSliderInput)

| State | Visual |
|-------|--------|
| Default | 4 (minimum — user sees minimum immediately) |
| Range | min=4, max=64, step=1 |
| Number input | Right-aligned, `w-24` — enforces min=4 constraint visually |
| Below minimum | VSAN_MAX_MIN_NODES error banner appears below the slider |

### Network speed button group (HostSpecsForm)

| State | Visual |
|-------|--------|
| Default active | 25 GbE |
| Active | `bg-blue-600 text-white border-blue-600` |
| Inactive | `bg-white dark:bg-gray-700 text-gray-700 border-gray-300 hover:border-blue-400` |
| 10 GbE + dedup enabled | DEDUP_NETWORK_SPEED warning appears in StorageConfigForm dedup section (severity: warning, yellow) |

### VsanMaxClusterCard visibility

| State | Condition |
|-------|-----------|
| Shown | `input.storageType === 'vsan-max'` |
| Hidden | Any other storage type — component unmounts from DOM (`v-if`, not `v-show`) |

### Validation states

| Code | Trigger | Severity | Component | Copy key |
|------|---------|----------|-----------|----------|
| VSAN_MAX_MIN_NODES | `vsanMaxStorageNodes < 4` | error | StorageConfigForm (input) + VsanMaxClusterCard (output) | `validation.vsanMaxMinNodes` |
| DEDUP_NETWORK_SPEED | `dedupEnabled && networkSpeedGbE < 25` | warning | StorageConfigForm dedup section | `validation.dedupNetworkSpeed` |

## Profile Label Display (VsanMaxClusterCard computed)

The `profileLabel` computed property in VsanMaxClusterCard formats the selected profile as `"{PROFILE} ({N} TB/node)"` for display in the card header row. Examples:

| Input value | Display label |
|-------------|--------------|
| `xs` | `XS (20 TB/node)` |
| `sm` | `SM (50 TB/node)` |
| `med` | `MED (100 TB/node)` |
| `lrg` | `LRG (150 TB/node)` |
| `xl` | `XL (200 TB/node)` |

This differs from the dropdown option format (em dash) to avoid the em dash in running prose.
