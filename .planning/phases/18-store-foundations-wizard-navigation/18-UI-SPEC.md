## Spacing Scale

Declared values (must be multiples of 4). All tokens are already established by the codebase; this phase adds no new spacing needs.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `gap-1.5`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`px-4`, `py-4`) |
| lg | 24px | Section padding (`px-6`, `py-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:

- Wizard step pill padding: `px-3 py-1.5` (12px horizontal, 6px vertical) — matches existing WizardStepper pattern, do not change.
- Header height: `py-3` + `text-lg` ≈ 56px — keep consistent with sticky header.
- Touch target for clickable completed step pills: minimum 32px height is satisfied by the existing `py-1.5 text-sm` pill. No special override needed (desktop-primary tool).

Source: verified against `WizardStepper.vue`, `App.vue`, `TopologySelector.vue` codebase inspection.

## Color

Existing project palette (Tailwind semantic classes). No custom hex values — all colors come from Tailwind v4 default palette. Dark mode variants are already established throughout the codebase.

| Role | Tailwind Token | Approx Hex | Usage |
|------|---------------|-----------|-------|
| Dominant (60%) | `gray-50` / `gray-950` | `#F9FAFB` / `#030712` | Page background (`bg-gray-50 dark:bg-gray-950`) |
| Secondary (30%) | `white` / `gray-900` | `#FFFFFF` / `#111827` | Cards, panels, header (`bg-white dark:bg-gray-900`) |
| Accent (10%) | `blue-600` | `#2563EB` | See reserved list below |
| Completed-state | `green-100` / `green-700` | `#DCFCE7` / `#15803D` | Completed wizard step pill background + text only |

**Accent (`blue-600`) reserved for:**

1. Active wizard step pill (`bg-blue-600 text-white border-blue-600`)
2. Primary action buttons: "Next" in WizardStepper, topology selection active state
3. "Start Sizing" CTA button on LandingView
4. Language switcher active locale chip
5. Border hover highlight on inactive topology buttons (`hover:border-blue-400`)

Accent must NOT be used for: step connectors, body text, section borders, completed steps, or informational labels.

**LandingView color usage:**

- Page background: `bg-gray-50 dark:bg-gray-950` (dominant, same as main app)
- Step summary cards: `bg-white dark:bg-gray-800 border border-gray-200 dark:border-gray-700 rounded-lg` (secondary)
- Step number badges: `bg-blue-600 text-white` (accent — exactly 3 badges, one per wizard step)
- CTA button: `bg-blue-600 text-white hover:bg-blue-700` (accent)

Source: `App.vue`, `WizardStepper.vue`, `TopologySelector.vue`, `LanguageSwitcher.vue` codebase inspection.

## Component Interaction Contract

### WIZARD-01: Completed Step Badge Click-Back

**Component:** `WizardStepper.vue`

**Current state:** Step pills have no `@click` handler. The completed state class is already defined as `bg-green-100 text-green-700 border-green-400 dark:bg-green-900 dark:text-green-300 dark:border-green-700`.

**Required changes:**

1. Add `cursor-pointer` class to completed step pill (condition: `ui.currentWizardStep > s.step`)
2. Add `hover:ring-2 hover:ring-green-400` on the completed pill for hover affordance
3. Add `@click` handler: fires `ui.setWizardStep(s.step)` only when `ui.currentWizardStep > s.step`
4. Active step pill (`ui.currentWizardStep === s.step`): no click handler, no cursor change
5. Future step pill (`ui.currentWizardStep < s.step`): no click handler, `cursor-not-allowed` is NOT added — pills remain neutral grey, no pointer cursor

**Resulting class structure for completed pill:**

```
bg-green-100 text-green-700 border-green-400
dark:bg-green-900 dark:text-green-300 dark:border-green-700
cursor-pointer hover:ring-2 hover:ring-green-400
```

**No animation or transition needed** for the click — `setWizardStep()` is synchronous, the content panel switches instantly via `v-show`.

### WIZARD-02: Landing View (LandingView.vue — new component)

**Scope:** Landing replaces only the `<main>` content area. The sticky `<header>` (containing the app title and `LanguageSwitcher`) remains visible. This allows Swiss multi-locale users to switch locale before entering the wizard.

**Source:** RESEARCH.md Assumption A1, Open Question 1 resolution.

**Layout:**

```
[sticky header — app title + LanguageSwitcher]          ← unchanged
[LandingView — fills min-h-[calc(100vh-56px)]]
  flex flex-col items-center justify-center gap-8
  bg-gray-50 dark:bg-gray-950 px-4 py-12
  ├─ [Product tagline — display text, centered]
  ├─ [Subtitle — body text, centered, max-w-lg]
  ├─ [3-step summary row — 3 cards side-by-side on md+, stacked on mobile]
  │   each card: bg-white dark:bg-gray-800 border rounded-lg p-4
  │   ├─ [step number badge — bg-blue-600 text-white w-8 h-8 rounded-full]
  │   ├─ [step title — text-base font-semibold]
  │   └─ [step description — text-sm text-gray-600 dark:text-gray-400]
  └─ ["Start Sizing" CTA button]
      bg-blue-600 text-white hover:bg-blue-700
      px-6 py-3 text-base font-semibold rounded-md
      transition-colors
```

**App.vue integration:**

```html
<LandingView v-if="ui.isLandingVisible" />
<div v-else class="min-h-screen bg-gray-50 dark:bg-gray-950 font-sans">
  <!-- existing main wizard layout unchanged -->
</div>
```

The header remains outside this `v-if/v-else` block, always rendered.

## Accessibility Contract

- Clickable completed step pills must receive `role="button"` or be converted to `<button>` elements. A bare `<div @click>` is not keyboard-accessible.
- CTA button on LandingView must be a `<button>` element (not a `<div>` or `<a>`).
- Step number badges on LandingView: include `aria-hidden="true"` on decorative number elements; step titles carry the semantic meaning.
- LandingView focus management: when landing is dismissed, focus should return to the first interactive element in the wizard (the topology selector). Use `nextTick` + manual focus after `dismissLanding()` if needed, or rely on natural DOM order (acceptable for v1 of this feature).
- Dark mode: all new classes must include `dark:` variants matching the existing pattern (`dark:bg-gray-800`, `dark:text-gray-300`, etc.).

## Print Behavior

LandingView must carry `print:hidden` so it never appears in print output. The landing state is ephemeral — if a user prints, they will have already dismissed the landing and be in the wizard layout, so this is belt-and-suspenders only.

```html
<div v-if="ui.isLandingVisible" class="print:hidden">
  <LandingView />
</div>
```
