All three charts use the **computed() prop pattern** (per RESEARCH.md Pattern 1, per instruction constraint #2). Do NOT use `shallowRef + triggerRef` for chart data — that pattern is only needed for imperative Chart.js instance access. Pass a `computed()` returning a NEW object reference to `:data` — vue-chartjs's built-in data watcher handles `chart.update()` automatically.

**3a — CoresChart.vue** (per VIZ-02):

Register: `ChartJS.register(Title, Tooltip, Legend, BarElement, CategoryScale, LinearScale)` — do this once at module level.

```typescript
const calc = useCalculationStore()
const { t } = useI18n()

const chartData = computed(() => ({
  labels: [t('results.charts.required'), t('results.charts.available')],
  datasets: [{
    label: t('results.charts.cores'),
    data: [calc.compute.totalCoresRequired, calc.compute.availableCores],
    backgroundColor: [
      calc.compute.totalCoresRequired > calc.compute.availableCores
        ? 'rgba(239,68,68,0.75)'   // red-500 — over capacity
        : 'rgba(20,184,166,0.75)', // teal-500 — safe
      'rgba(100,116,139,0.4)',     // slate-500 — available bar
    ],
  }]
}))

const chartOptions = {
  responsive: true,
  maintainAspectRatio: false,
  plugins: { legend: { display: false }, tooltip: { enabled: true } },
  scales: { y: { beginAtZero: true } }
}
```

Template: `<div class="bg-white rounded-lg shadow-sm border border-gray-200 p-4 h-48 relative"><Bar :data="chartData" :options="chartOptions" /></div>`
