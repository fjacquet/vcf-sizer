### Human Verification Required

#### 1. Input Panel Hidden in Print Preview

**Test:** Open the app in a browser, populate inputs, then trigger the browser print dialog (Cmd+P on macOS / Ctrl+P on Windows/Linux).
**Expected:** The print preview shows only the results panel. The "VCF 9.x Sizing Calculator" page header, language switcher, and all input forms on the left pane are completely absent from every page.
**Why human:** CSS `print:hidden` (i.e., `@media print { display: none }`) suppression cannot be verified by Vitest — it requires a layout engine.

#### 2. Header and Footer on Every Printed Page

**Test:** In the same print preview, inspect multiple pages if the results span more than one page.
**Expected:** Every page shows "VCF 9.x Sizing Report" on the left side of the header and today's date on the right. Every page shows "Generated by VCF Sizer" centered in the footer.
**Why human:** CSS `position: fixed` in print media is rendered by the browser's print layout engine; Vitest's jsdom does not simulate fixed positioning across paginated pages.

#### 3. Chart Fallback Tables Replace Canvas Bars

**Test:** In the print preview, scroll to the Cores, RAM, and Storage chart areas.
**Expected:** The bar chart canvas is not visible. In its place, a plain bordered table is shown with the correct numeric values matching what the chart would have displayed on screen.
**Why human:** `print:hidden` / `print:table` CSS toggling requires a browser layout engine; Vitest cannot assert display property changes under `@media print`.
